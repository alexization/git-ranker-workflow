# Agent Readiness Scorecard

이 문서는 `GRW-02`부터 사용하는 readiness 기준선 평가 축과 점수 규칙을 정의한다.

## 평가 항목

| 항목 | 질문 |
| --- | --- |
| 문서화 | 새 Agent가 공식 문서만 읽고 목적, 범위, 제약, 주요 흐름을 이해할 수 있는가 |
| 테스트 | 핵심 동작을 검증하는 자동화 테스트가 존재하고 실제로 돌릴 수 있는가 |
| 정적 가드레일 | lint, type check, coverage, architecture rule, CI 같은 정적 품질 장치가 있는가 |
| API 계약 | OpenAPI, generated schema, 단일 계약 타입처럼 API 인터페이스를 고정하는 자산이 있는가 |
| 운영 재현성 | 로컬/CI/하네스에서 같은 방식으로 기동, reset, build, env 주입을 재현할 수 있는가 |
| 관측 가능성 | health, metrics, logs, dashboard, tracing 중 필요한 신호를 수집할 수 있는가 |
| Agent 탐색 가능성 | 저장소 구조, 진입 문서, 명령, 핵심 경로가 Agent 기준으로 빠르게 탐색 가능한가 |

## 점수 기준

| 점수 | 의미 |
| --- | --- |
| `1` | 핵심 자산이 없거나 바로 작업을 막는 blocker가 있다 |
| `2` | 자산은 일부 있으나 갭이 커서 반복 작업이나 하네스 적용이 불안정하다 |
| `3` | 기본 사용은 가능하지만 후속 작업 전에 메워야 할 중요한 갭이 남아 있다 |
| `4` | 대부분 준비됐고 남은 갭이 제한적이다 |
| `5` | 문서, 검증, 재현성이 안정적으로 갖춰져 바로 하네스 반복 실행이 가능하다 |

## `N/A` 규칙

- 저장소 역할상 해당 항목이 본질적으로 적용되지 않으면 `N/A`로 적는다.
- `N/A` 항목은 overall score 평균 계산에서 제외한다.
- 현재 기준 예시: `git-ranker-workflow`의 `API 계약` 항목

## `Low / Medium / High` 규칙

- `Low / Medium / High`는 readiness 자체가 아니라 `현재 남아 있는 핵심 리스크 수준`을 뜻한다.
- `Low`: 즉시 막히는 문제는 없고 후속 개선이 중심인 상태
- `Medium`: 다음 작업에서 다뤄야 할 구조적 갭이 있으나 우회 가능성이 있는 상태
- `High`: build, test, 계약, 런타임 중 하나 이상이 직접 blocker로 남아 있는 상태

## Overall Score 계산

- 저장소별 overall score는 적용 가능한 항목의 점수를 평균 내고 반올림해 `1~5` 정수로 적는다.
- overall score와 별도로 `핵심 리스크 수준`과 `핵심 리스크 요약`을 반드시 함께 남긴다.

## 증거 규칙

- 각 평가는 실제 확인한 명령과 확인 날짜를 함께 남긴다.
- 명령 전체 출력은 필요할 때만 별도 참고 문서에 보관하고, 기준선 문서에는 핵심 결과만 요약한다.
- 문서 전용 저장소는 브라우저/로그/메트릭 artifact가 없어도 되지만, 무엇을 확인했고 무엇이 비어 있는지는 명확히 적는다.
